Почитал и собрал отличия отличия,
- CentOS - Redhat, Ubuntu - Debian.
- ни один администратор в системе Ubuntu не знает пароль root (он генерируется случайным образом), а привилегии root предоставляются с помощью команды sudo.
- Ubuntu использует apt с пакетами DEB, а CentOS использует менеджер пакетов YUM с пакетами RPM.
- Ubuntu имеет тенденцию быть менее консервативным, чем CentOS, что означает, что его репозитории содержат более свежие пакеты, тогда как пакеты CentOS, как правило, фокусируются на исправлениях безопасности и стабильности / согласованности.
- в DEBIAN  вы можете удалить пакет, не оставляя лишних в вашей системе
- RHEL  - лучший выбор, если безопасность действительно важна (потому что CentOS не будет устанавливать патчи CVE, даже если у yum есть опция).
- Простое упправление кластерами Hadoop на CentOS, отличная докуменация о RedHat Access для RHEL / CentOS. 
- Для настольных компьютеров Ubuntu может быть «лучше»
- CentOS поддерживает версии дольше. 
- В CentOS есть одна особенность, а именно коллекции программного обеспечения.
- В CentOS могут быть «устаревшие» инструменты (такие как python, ruby, perl, GCC, Apache и т. Д.). Но это стабильная база, и можно установить более новые apache, ruby, python, git… на коробку * без * воздействия на базовое программное обеспечение. Это очень, очень удобно.

$ scl включить rh-python35 bash
$ python --version
Python 3.5.1
Что еще лучше, так это то, что вы можете сложить.

$ scl enable rh-pyton35 rh-nodejs8 bash

Это действительно круто.
И это не приводит к никаким изменениям на системном уровне, поэтому все по-прежнему работает должным образом, обновляется и т. Д. 

- CentOS. Для разработки есть devtoolset, предоставляющий вам доступ к различным компиляторам и инструментам.
- Набор инструментов разработчика 7 - Набор инструментов разработчика Red Hat
- Есть также контейнерные версии этих вещей, которые можно легко запустить на CentOS. Делать то же самое в Ubuntu на самом деле не эквивалентно. 
- Можно запускать виртуальные машины Ubuntu с CentOS, делать снимок и вернуться в прошлое, если происходит серьезный беспорядок.
- На самом деле, намного сложнее удалить пакеты в Ubuntu, когда дела пошли плохо, чем в CentOS.
- Конечно, для игр на Linux ответ - Ubuntu. Драйверы и т. Д. Более сложны для других дистрибутивов.
- Профессионализм CentOS / RHEL можно оценить уже после использования. 
- Ubuntu может быть модным автомобилем. Но CentOS / RHEL - это рабочие лошадки / фабрики / поезда.
